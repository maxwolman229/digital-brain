SDK for self-improving plant chatbots

A first-pass architecture: easy install, customer-controlled infra, conversation mining into governed operational knowledge.

Proposed data flow

Existing Chatbot

User, assistant, RAG citations, tool calls, feedback, session metadata.

MD1 Context SDK

Server-side middleware captures events, redacts sensitive fields, scores signals, and forwards to a local or managed collector.

Customer/MD1 Store

Conversation ledger, knowledge candidates, domain map, usage analytics, governance queue.

Improvement loop

Mine tacit context → propose rules/assertions/SOP deltas → route to SME review → publish back to RAG index, knowledge bank, or MCP/tool context.

A

Drop-in SDK First

Fastest implementation. Add middleware around chat requests/responses and stream normalized events to the context engine.

B

Proxy/Gateway Mode

Best visibility. Route LLM/RAG calls through an observability gateway when teams cannot alter chatbot internals deeply.

C

MCP/Tool Integration

Most agent-native. Chatbots call MD1 as a context and feedback tool, useful when their stack already exposes MCP/tools.

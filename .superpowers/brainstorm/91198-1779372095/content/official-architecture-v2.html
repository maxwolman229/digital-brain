MD1 Context SDK System Architecture

Managed-cloud deployment for a customer with an existing manufacturing chatbot or RAG assistant. The customer keeps their chatbot, RAG stack, documents, auth, and model path; MD1 mines normalized conversation telemetry into reviewed operating knowledge.

Customer Environment

Existing systems. MD1 does not replace the chatbot, retriever, document store, or LLM path.

Plant Chatbot UI

Operators, engineers, supervisors, and support teams ask questions in the customer’s existing assistant.

requests
Chatbot Application Server

Owns auth, sessions, prompts, tools, retrieval calls, model calls, and response handling.

uses
Existing RAG Stack

Retriever, vector database, chunks, metadata, SOPs, manuals, tickets, MES/ERP-derived context.

LLM Provider or Model Gateway

OpenAI, Anthropic, Azure, Bedrock, internal model gateway, or agent runtime.

MD1 SDK runs here, inside the chatbot server.
MD1 Context SDK

Server-side middleware captures normalized events: messages, citations, retrieval context, feedback, corrections, user/domain metadata, tool calls, and outcomes.

MD1 Managed Cloud

MD1-owned context engine, tenant isolation, mining workflow, and review surface.

Event Ingestion API

Receives SDK events through server-side keys, redaction hooks, allowlists, and tenant routing.

stores
Conversation Ledger

Structured record of sessions, messages, retrieved sources, responses, feedback, sentiment, and provenance.

mines
Knowledge Mining Engine

Extracts adjacent operating context, disagreement, process exceptions, missing-source demand, sentiment, expert patterns, and candidate knowledge.

routes
Human Review Queue

SMEs approve, reject, edit, merge, verify, and comment on candidates with source conversation attached.

Analytics + RAG Improvement Feed

Shows domain failures, trust signals, unanswered clusters, bad citations, stale references, weak chunks, and suggested fixes.

Approved Destinations

Nothing publishes automatically. Approved outputs go where the customer wants value first.

RAG / Vector Index

Default fast path. Approved context improves future chatbot answers and retrieval coverage.

SOP / Wiki / Document Systems

SharePoint, Confluence, Google Drive, controlled document workflows, or internal knowledge portals.

MD1 Knowledge Bank

Optional expansion into lifecycle status, verification, comments, versions, links, graph, archive, and health dashboard.

Tickets / Webhooks / Data Warehouse

Jira, ServiceNow, Linear, custom API, CSV export, BI tools, or analytics lake.

Approved knowledge closes the loop back into the customer’s AI assistant.
Improved Chatbot

Better local context, fewer repeated failures, stronger answers, clearer ownership of knowledge gaps.

What crosses the boundary

  • Normalized conversation events
  • Retrieval/citation metadata
  • User role, domain, plant area
  • Feedback, corrections, outcomes

Security controls

  • Server-side SDK key
  • Field allowlists and redaction
  • Tenant isolation and audit logs
  • Retention controls; VPC/customer-hosted path later

Key integration factors

  • Can the chatbot server be modified?
  • Can retrieval context and citations be captured?
  • Is there MCP/tool-call visibility?
  • Where should approved knowledge publish first?

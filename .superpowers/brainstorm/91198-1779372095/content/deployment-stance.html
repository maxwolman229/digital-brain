Recommended deployment: MD1 managed cloud first

The SDK is lightweight; MD1 runs the context engine, dashboards, candidate review, and RAG improvement feed.

v1 managed architecture

Customer chatbot stack

Chat UI, server endpoint, RAG retriever, vector DB, documents, LLM provider, auth/user roles.

events →

MD1 Context Cloud

Event ingestion, PII controls, conversation ledger, signal extraction, analytics, candidate queue, RAG improvement feed.

Outbound artifacts

CSV/API/webhooks into Jira, Confluence, SharePoint, vector DB, or MD1 Knowledge Bank.

Security story

Server-side key, redaction hooks, configurable fields, tenant isolation, audit log, data retention controls.

Later options

Customer-hosted collector, VPC deploy, or derived-signals-only mode for stricter sites.

Why managed first

  • Fastest integration and sales proof
  • Centralizes improvement models
  • Lets us iterate on dashboards and mining logic
  • Clean expansion into MD1 governance

Risks to answer

  • Raw conversation data may be sensitive
  • Plants may restrict cloud egress
  • Auth and user identity mapping vary by chatbot
  • RAG stack visibility may be incomplete
